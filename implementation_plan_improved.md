# 材料晶体数据库前后端实现计划（改进版）

## 基于 00-MASTER-PLAN.md 的改进分析

### 核心改进点总结
1. **架构现代化**: Redis 缓存层、Ant Design ProTable、TanStack Query
2. **功能增强**: 元素周期表选择器、3D 可视化、材料对比、URL 状态持久化
3. **性能优化**: 物化视图、字段投影、虚拟滚动、ETag 缓存

### 潜在实现问题与解决方案

| 改进项目 | 潜在问题 | 风险评估 | 解决方案 |
|----------|----------|----------|----------|
| **Ant Design 替代 MUI** | 开发团队需要学习新组件库 | 低 | Ant Design 文档完善，ProTable 组件专门为数据密集型场景优化，长期收益高 |
| **TanStack Query 状态管理** | 学习曲线，可能过度复杂 | 低 | 对于服务端状态管理（分页、缓存、同步）天然合适，减少自定义代码 |
| **Redis 缓存层** | 增加基础设施复杂度 | 中 | 初期可作为可选组件，使用内存缓存回退，后期需要时再部署完整 Redis |
| **Three.js 3D 可视化** | 技术复杂度高，开发时间长 | 高 | 使用简化库如 3dmol.js 或 crystaltoolkit，或作为二期功能 |
| **字段投影 API** | API 设计复杂，可能影响性能 | 中 | 使用 GraphQL-like 查询参数，在服务端实现字段选择，默认返回核心字段 |
| **物化视图** | 需要数据库管理权限，数据更新延迟 | 中 | 作为性能优化选项，在统计查询慢时再实现 |
| **URL 状态持久化** | URL 可能过长，浏览器兼容性 | 低 | 使用 URL 压缩算法（如 lz-string），设置长度限制 |

## 实施阶段调整（基于改进要求）

### 阶段1: 项目初始化与环境搭建 (0.5天)
1. **创建改进的项目结构** (按 00-MASTER-PLAN.md)
2. **后端环境**
   - Python 虚拟环境 + `requirements.txt` (包含 Redis 客户端)
   - 核心依赖: `fastapi`, `sqlalchemy`, `asyncpg`, `pydantic`, `uvicorn`, `redis`
3. **前端环境**
   - Vite + React + TypeScript 项目初始化
   - 安装 Ant Design 5 + TanStack Query v5 + axios
   - 路由配置 (React Router v6)

### 阶段2: 后端 API 开发 (2天，含改进功能)
1. **数据库模型定义** (不变)
2. **Pydantic Schema 设计**
   - **改进**: 实现字段投影机制，支持 `?fields=id,formula,band_gap`
   - 分层次数据模型 + 按需加载字段
3. **API 端点实现** (新增/改进)
   | 端点 | 方法 | 功能 | 改进点 |
   |------|------|------|--------|
   | `/materials` | GET | 材料列表 | 支持字段投影，Redis 缓存 |
   | `/materials/{id}` | GET | 材料详情 | 支持字段投影，ETag 缓存 |
   | `/materials/search` | GET | 高级搜索 | URL 参数持久化支持 |
   | `/materials/compare` | POST | 材料对比 | **新增**: 多材料对比端点 |
   | `/materials/elements` | GET | 元素统计 | 物化视图优化 |
   | `/materials/statistics` | GET | 数据统计 | Redis 缓存统计结果 |
   | `/materials/periodic-table` | GET | 元素周期表数据 | **新增**: 支持前端交互选择器 |

4. **查询优化**
   - 利用现有数据库索引
   - 实现 Redis 缓存层 (可配置开关)
   - 支持 ETag / If-None-Match 条件请求

### 阶段3: 前端核心框架 (1.5天)
1. **项目架构**
   - React Router 路由配置
   - **改进**: TanStack Query Provider 配置
   - API 服务层封装 (基于 TanStack Query)
2. **状态管理策略**
   - 服务端状态: TanStack Query (自动缓存、同步、分页)
   - 客户端状态: React useState/useReducer (UI 状态)
   - **改进**: URL 状态同步 (React Router + query-string)

### 阶段4: 前端界面开发 (2.5天)
1. **核心页面** (使用 Ant Design 组件)
   - **材料列表页**: Ant Design ProTable + 虚拟滚动 + 字段选择器
   - **材料详情页**: 标签页布局 + 按需加载详细字段
   - **高级搜索页**: 改进的过滤面板 + 元素周期表选择器
   - **对比分析页**: **新增**: 并排比较表格/图表
2. **特色组件**
   - `PeriodicTableSelector`: 交互式元素周期表
   - `MaterialComparison`: 多材料对比组件
   - `StructureViewer`: 3D 晶体结构可视化 (可选)

### 阶段5: 功能增强与优化 (1.5天)
1. **数据可视化**
   - 使用 Recharts 或 ECharts
   - 带隙分布、元素组成、相关性分析
2. **性能优化**
   - 前端: 虚拟滚动、搜索防抖、按需加载字段
   - 后端: Redis 缓存策略、查询优化、物化视图
3. **用户体验**
   - URL 状态持久化 (可分享的搜索链接)
   - 响应式设计 (Ant Design 内置)
   - 加载状态和错误处理

### 阶段6: 测试与部署 (1天)
1. **测试策略**
   - 后端: pytest (包含缓存层测试)
   - 前端: Jest + React Testing Library
   - E2E: Playwright (测试完整用户流程)
2. **容器化部署**
   - Docker Compose (PostgreSQL + Redis + Backend + Frontend)
   - 环境变量配置 (Redis 可选)
   - Nginx 反向代理配置

## 风险评估与缓解

### 高风险项目
1. **Three.js 3D 可视化**
   - **风险**: 开发时间长，技术复杂度高
   - **缓解**: 使用简化库 (3dmol.js)，或作为二期功能
   - **备选**: 提供晶体结构数据下载，用户用专业工具查看

2. **Redis 缓存一致性**
   - **风险**: 缓存失效策略复杂，可能返回旧数据
   - **缓解**: 实现 TTL + 手动失效，关键数据实时查询
   - **备选**: 初期使用内存缓存，后期迁移到 Redis

### 中风险项目
1. **字段投影 API**
   - **风险**: API 复杂度增加，维护困难
   - **缓解**: 使用标准 GraphQL-like 语法，实现核心字段投影
   - **测试**: 充分测试不同字段组合的查询性能

2. **物化视图维护**
   - **风险**: 数据更新延迟，需要定期刷新
   - **缓解**: 使用 PostgreSQL 自动刷新，或定时任务
   - **监控**: 监控物化视图刷新状态

### 低风险项目
1. **Ant Design 迁移**
   - **风险**: 学习新组件库
   - **缓解**: Ant Design 文档完善，社区活跃
2. **TanStack Query 集成**
   - **风险**: 状态管理概念变化
   - **缓解**: TanStack Query 是 React 生态标准，大量教程
